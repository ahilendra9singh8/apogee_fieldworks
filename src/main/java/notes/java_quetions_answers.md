1. Why is Java Platform Independent?

Because Java code is compiled into bytecode, which runs on the JVM (Java Virtual Machine) —
and JVM is available for all platforms (Windows, Linux, Mac, etc.).

2. What is Bytecode?

Bytecode is an intermediate code generated by the Java compiler (.class file).
It’s not machine-dependent — the JVM interprets it into native machine code.

3. What are the main features of Java?

| **Simple**               | Easy syntax similar to C/C++        |

12. What is this keyword?

this refers to current object inside a method or constructor.

✅ Uses of this:
1️. Differentiate instance & local variables
2️. Call another constructor in same class
3️. Pass current object as argument

Example 1 – differentiate variables

class Demo {

    int age;

    Demo(int age) {
        this.age = age;   // 'this' refers to instance variable
    }
}

Example 2 – call another constructor

class A {

    A() {
        this(10);
        System.out.println("Default constructor");
    }
    A(int x) {
        System.out.println("Parameterized constructor: " + x);
    }

    public static void main(String[] args) {
        new A();
    }
}

Example 3 – pass current object

class Test {

    void display(Test t) {
        System.out.println("Method called using this");
    }
    void call() {
        display(this);
    }
    public static void main(String[] args) {
        new Test().call();
    }
}


13. What is super keyword?

super refers to the parent class (superclass).

✅ Uses of super:
1️. Access parent class variable or method
2️. Call parent constructor

Example:

class Parent {

    int num = 100;
    Parent() {
        System.out.println("Parent constructor");
    }
    void show() { System.out.println("Parent method"); }
}

class Child extends Parent {

    int num = 200;

    Child() {
        super(); // calls Parent constructor
        System.out.println("Child constructor");
    }

    void display() {
        System.out.println("Parent num: " + super.num);
        System.out.println("Child num: " + num);
        super.show();
    }

    public static void main(String[] args) {
        Child c = new Child();
        c.display();
    }
}

18. finalize() method

    }
}



22. Why String is Immutable in Java

1. Once a String object is created, its value cannot be changed.
2. Every modification creates a new object in the String pool.
3. Immutability ensures security, thread-safety, and caching.

Example:-

String s1 = "Hello";
String s2 = s1;       // both point to same object
s1 = s1 + " World";   // new object created
System.out.println(s1); // Hello World
System.out.println(s2); // Hello


23. Ways to Create String

String s1 = "Java";              // using string literal (stored in String Pool)
String s2 = new String("Java");  // using new keyword (stored in heap)

Difference:
Literal → checks pool; reuse if exists.
new → always creates a new object.

🔹 1. Program: Demonstrate Method Overloading

class Calculator {

    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}

public class Main {

    public static void main(String[] args) {
        Calculator c = new Calculator();
        System.out.println(c.add(5, 10));        // int version
        System.out.println(c.add(2.5, 3.5));     // double version
        System.out.println(c.add(1, 2, 3));      // 3-arg version
    }
}

✅ Concept: Compile-time polymorphism (method overloading).
Same name → different parameter list.

🔹 3. Program: Dynamic Method Dispatch Example

Dynamic Method Dispatch = runtime decision → which method to call.

class A {

    void show() {
        System.out.println("Show from A");
    }
}

class B extends A {

    void show() {
        System.out.println("Show from B");
    }
}

class C extends A {

    void show() {
        System.out.println("Show from C");
    }
}

public class Test {

    public static void main(String[] args) {
        A obj;         // reference of parent
        obj = new B(); // object of B
        obj.show();    // Show from B

        obj = new C();
        obj.show();    // Show from C
    }
}

✅ Concept: Object type decides which method executes (not reference type).

🔹 9. Program: Singleton Class in Java

A Singleton class allows only one object to be created.

class Singleton {

    private static Singleton instance; // single instance

    private Singleton() {} // private constructor

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }

    void showMessage() {
        System.out.println("Singleton instance running...");
    }
}

public class Test {

    public static void main(String[] args) {

    }
}

8. Custom Exception (User-defined Exception)

You can create your own exception class using extends Exception.

class InvalidAgeException extends Exception {

    public InvalidAgeException(String msg) {
        super(msg);
    }
}

public class Main {

    static void validate(int age) throws InvalidAgeException {
        if (age < 18)
            throw new InvalidAgeException("Age must be 18+");
        else
            System.out.println("Welcome to vote");
    }

    public static void main(String[] args) {
        try {
            validate(15);
        } catch (InvalidAgeException e) {
            System.out.println("Caught Exception: " + e.getMessage());
        }
    }
}


✅ Output:
Caught Exception: Age must be 18+

✅ Concept:
Useful when you want custom error messages or domain-specific rules.

1.Multithreading

Multithreading means executing multiple threads simultaneously to make the program faster and more efficient.
Each thread is a lightweight sub-process that runs independently.

🧩 Real-life example:
A web browser downloads a file, plays music, and lets you type at the same time — each activity runs in a separate thread.


2️. Thread vs Process

| Feature       | Process             | Thread                                      |
| ------------- | ------------------- | ------------------------------------------- |
| Definition    | Independent program | Subset (smallest unit) of a process         |
| Memory        | Separate memory     | Shared memory of process                    |
| Communication | Slow (via IPC)      | Fast (shared variables)                     |
| Example       | MS Word             | Spell-check + autosave + UI refresh threads |


3️. How to Create a Thread in Java

There are two main ways to create a thread:

🧮 Method 1: By Extending Thread Class

class MyThread extends Thread {

    public void run() {
        System.out.println("Thread is running using Thread class");
    }

    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        t1.start();  // starts new thread
    }
}

✅ Explanation:
run() contains the code to execute in a thread.
start() starts a new thread and internally calls run().


🧮 Method 2: By Implementing Runnable Interface

class MyRunnable implements Runnable {

    public void run() {
        System.out.println("Thread is running using Runnable interface");
    }

    public static void main(String[] args) {
        MyRunnable obj = new MyRunnable();
        Thread t1 = new Thread(obj);
        t1.start();
    }
}

✅ Preferred Approach
Because Java supports multiple inheritance via interfaces, using Runnable gives more flexibility.

8. Synchronization

When multiple threads access a shared resource, race conditions can occur.
To prevent this, we use synchronization.

🧩 Example (Without Synchronization) ❌

class Table {

    void printTable(int n) {
        for (int i = 1; i <= 5; i++) {
            System.out.println(n * i);
            try { Thread.sleep(400); } catch (Exception e) {}
        }
    }
}

class MyThread1 extends Thread {

    Table t;
    MyThread1(Table t) { this.t = t; }
    public void run() { t.printTable(5); }
}

class MyThread2 extends Thread {

    Table t;
    MyThread2(Table t) { this.t = t; }
    public void run() { t.printTable(100); }
}

public class Main {

    public static void main(String[] args) {
        Table obj = new Table();
        MyThread1 t1 = new MyThread1(obj);
        MyThread2 t2 = new MyThread2(obj);
        t1.start();
        t2.start();
    }
}

⚠️ Output gets mixed (race condition).


✅ Example (With Synchronization)

class Table {

    synchronized void printTable(int n) {
        for (int i = 1; i <= 5; i++) {
            System.out.println(n * i);
            try { Thread.sleep(400); } catch (Exception e) {}
        }
    }
}

✅ Now both threads run one at a time safely.



9. Inter-Thread Communication (wait(), notify(), notifyAll())

Used when threads need to communicate — for example, producer-consumer problems.

Example:-

class Shared {
    boolean ready = false;

    synchronized void produce() {
        System.out.println("Producing...");
        ready = true;
        notify();
    }

    synchronized void consume() throws InterruptedException {
        while (!ready)
            wait();
        System.out.println("Consuming...");
    }
}

public class Main {
    public static void main(String[] args) throws InterruptedException {
    
        Shared s = new Shared();

        Thread producer = new Thread(() -> s.produce());
        Thread consumer = new Thread(() -> {
            try { s.consume(); } catch (Exception e) {}
        });

        consumer.start();
        Thread.sleep(1000);
        producer.start();
    }
}

7. HashMap Internal Working (Simple & Accurate)

⭐ When you call map.put(key, value):

Step 1 → Create hashcode
int hash = key.hashCode();

Step 2 → Create index
Java uses:
index = (hash) & (n - 1);   // n = table size e.g., 16

Step 3 → Check bucket
-> HashMap has an array called table
-> Default size = 16, threshold = 12 (16 * 0.75)

Step 4 → If bucket is empty → insert Node
Step 5 → If bucket already has a node (hash collision)
=> Java compares hash
=> Then compares key using equals()
→ If same key → replace value
→ If different key → add new node → linked list
→ If linked list size > 8 → convert to Red-Black Tree

⭐ If key is null
-> Never computes hashcode
-> Always stored at bucket 0

✔ Simple Internal Working Example

map.put("A", 10)
hash("A") → 64532
index = 64532 & 15  → 4
Bucket[4] is empty → store Node("A",10)

map.put("B", 20)
hash("B") → 64538
index = 64538 & 15 → 4  (same index → collision)

Bucket 4 has Node(A,10)
→ compare hash
→ compare equals(keys)
→ keys are different → append to linked list

## HashMap has an array called table those 0 to 16 index and threshold value is 12, when use .put(key, value) method so create a hashcode than        index using hashcode these according index match table index insert key,value on perticular index.

-> Use bounded generics: <T extends Number> to limit to numeric types.

What is flatMap()?

-> Used when one element produces multiple elements
-> It flattens nested structures
-> One input → many outputs → flattened

import java.util.*;

            .forEach(System.out::println);
    }
}

| Output: Stream<Stream<T>>      | Output: Stream<T>      |


#4. ConcurrentHashMap
What is ConcurrentHashMap?
-> Thread-safe Map
-> Allows multiple threads to read & write safely
-> Faster than Hashtable
-> Used in multi-threaded applications

Why not HashMap?
-> ❌ HashMap is NOT thread-safe
-> ❌ Multiple threads can cause data inconsistency

Why not Hashtable?
-> ✔ Thread-safe
-> ❌ Very slow (locks entire map)


ConcurrentHashMap internally
-> Divides map into segments
-> Locks only a small part, not whole map
-> Allows concurrent read + write


## Simple Example — ConcurrentHashMap

import java.util.concurrent.*;
public class ConcurrentHashMapDemo {

    public static void main(String[] args) {
        ConcurrentHashMap<Integer, String> map =
                new ConcurrentHashMap<>();

        map.put(1, "Java");
        map.put(2, "Spring");

        map.forEach((k, v) -> System.out.println(k + " " + v));
    }
}

Output
1 Java
2 Spring


##Multi-Thread Example

import java.util.concurrent.*;
public class CHMDemo2 {

    public static void main(String[] args) {

        ConcurrentHashMap<Integer, String> map = new ConcurrentHashMap<>();

        Runnable r1 = () -> map.put(1, "Thread1");
        Runnable r2 = () -> map.put(2, "Thread2");

        new Thread(r1).start();
        new Thread(r2).start();
    }
}

✔ No exception
✔ Safe concurrent modification
